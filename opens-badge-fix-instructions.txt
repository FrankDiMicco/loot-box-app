================================================================================
BUG FIX: BOXCARD SHOWS "UNLIMITED OPENS" ON SHARED BOXES WITH LIMITS
================================================================================
File: index.html

PROBLEM
--------------------------------------------------------------------------------
Shared/group boxes with maxPulls set (e.g. 3) show "Unlimited Opens" on the
BoxCard instead of "1/3 Opens Left". The maxPulls value is being lost during
the shared box data refresh cycle.

ROOT CAUSE
--------------------------------------------------------------------------------
In the App component, when shared boxes are refreshed from Firestore on load,
the update merges fresh data but only explicitly copies pullHistory, items,
and name. maxPulls (and maxPullsPerUser) from the Firestore document are NOT
carried over, so they can be lost if the local reference is stale.

Find this code block in the App component's refresh/load logic (inside the
useEffect or loadBoxes function that fetches shared box data):

  const updatedBox = {
    ...box,
    pullHistory: freshBox.pullHistory || [],
    items: freshBox.items || box.items,
    name: freshBox.name || box.name,
  };

FIX: Add maxPulls and maxPullsPerUser to the merge:

  const updatedBox = {
    ...box,
    pullHistory: freshBox.pullHistory || [],
    items: freshBox.items || box.items,
    name: freshBox.name || box.name,
    maxPulls: freshBox.maxPulls != null ? freshBox.maxPulls : box.maxPulls,
    maxPullsPerUser: freshBox.maxPullsPerUser != null ? freshBox.maxPullsPerUser : box.maxPullsPerUser,
    hideContents: freshBox.hideContents != null ? freshBox.hideContents : box.hideContents,
    hideOdds: freshBox.hideOdds != null ? freshBox.hideOdds : box.hideOdds,
    expiresAt: freshBox.expiresAt != null ? freshBox.expiresAt : box.expiresAt,
    boxImageId: freshBox.boxImageId || box.boxImageId,
    allowParticipantSharing: freshBox.allowParticipantSharing != null ? freshBox.allowParticipantSharing : box.allowParticipantSharing,
  };

NOTE: Use `!= null` (not `!== null`) to catch both null and undefined, so
that a Firestore value of 0 or false is still respected.

ALSO CHECK: The real-time listener (subscribeToSharedBox) in BoxOpener
updates pullHistory but the BoxCard on the home screen may not re-render
with the latest data. Make sure the boxes state in the App component is
updated when a shared box's data changes. If boxes are only refreshed on
initial load, the BoxCard badges will be stale until the user navigates
away and back.
