================================================================================
FIX: Stack odds/history vertically on mobile + make both collapsible
================================================================================

CURRENT STATE
-------------
In BoxOpener, odds and pull history sit side by side in a 2-column grid:

    gridTemplateColumns: box.hideOdds ? '1fr' : '1fr 1fr'

This is cramped on mobile and both panels compete for space.


DESIRED BEHAVIOR
----------------
- MOBILE: Stack vertically -- odds on top, pull history below
- DESKTOP: Keep side-by-side 2-column layout (no change)
- BOTH: Each panel is collapsible with a tap-to-toggle header
- Odds panel: expanded by default (useful reference while pulling)
- Pull history: collapsed by default (grows long over time)
- The collapse toggle should be a chevron arrow that rotates


CHANGES IN index.html - BoxOpener component
--------------------------------------------

CHANGE 1: Add collapse state variables

Near the top of the BoxOpener component (with the other useState calls),
add:

    const [oddsExpanded, setOddsExpanded] = useState(true);
    const [historyExpanded, setHistoryExpanded] = useState(false);

Use the useIsMobile() hook (from mobile-ux-improvements.txt):

    const isMobile = useIsMobile();


CHANGE 2: Make the grid responsive

Find the div with:
    gridTemplateColumns: box.hideOdds ? '1fr' : '1fr 1fr'

Replace the entire style object with:

    display: 'grid',
    gridTemplateColumns: isMobile ? '1fr' : (box.hideOdds ? '1fr' : '1fr 1fr'),
    gap: isMobile ? '1rem' : '1.5rem',

CHANGE 5: Auto-expand pull history after first pull

It's nice to auto-open the history panel when the user gets their
first result so they can see it logged. Add this logic inside the
handleOpen function, right after a successful pull is recorded:

    // After adding the pull to history:
    if (!historyExpanded) {
      setHistoryExpanded(true);
    }

This way:
- First visit: history is collapsed (nothing to see yet)
- After first pull: auto-expands to show the result
- User can collapse it again if they want
- Subsequent pulls keep it open unless user collapsed it

Actually, to avoid forcing it open if the user deliberately collapsed
it, use a ref to track if user has manually toggled:

    const userToggledHistory = useRef(false);

    // In the onClick for history toggle:
    onClick={() => {
      setHistoryExpanded(!historyExpanded);
      userToggledHistory.current = true;
    }}

    // In handleOpen after a pull:
    if (!historyExpanded && !userToggledHistory.current) {
      setHistoryExpanded(true);
    }


CHANGE 6: Collapsible header touch target

    padding: '0.25rem 0'

on the clickable div wrapper.
